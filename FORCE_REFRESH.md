## 🎯 为什么看不到更新？

**原因：浏览器缓存了旧的页面**

VitePress 在开发模式下会进行热更新，但浏览器可能会缓存 HTML 和 JavaScript 文件。

## 📱 解决方案（按顺序尝试）

### 方法 1: 强制刷新（最有效）

在你的浏览器中访问 http://localhost:5173，然后：

**Windows/Linux:**
```
Ctrl + Shift + R
```
或
```
Ctrl + F5
```

**Mac:**
```
Cmd + Shift + R
```

### 方法 2: 清除浏览器缓存

**Chrome/Edge:**
1. 按 `F12` 打开开发者工具
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

**Firefox:**
1. 按 `Ctrl + Shift + Delete`
2. 选择"缓存"
3. 点击"立即清除"

**Safari:**
1. 按 `Cmd + Option + E` 清除缓存
2. 按 `Cmd + R` 刷新页面

### 方法 3: 使用无痕模式

打开浏览器的无痕/隐私模式，访问 http://localhost:5173

如果无痕模式下能看到新导航栏，说明确实是缓存问题。

### 方法 4: 添加时间戳参数

访问时添加随机参数破坏缓存：
```
http://localhost:5173/?t=202604091752
```

### 方法 5: 直接访问新页面测试

即使导航栏没显示，也可以直接访问新页面验证功能：

- **RAG 知识库**: http://localhost:5173/rag
- **情感分析**: http://localhost:5173/emotion
- **AI 俄罗斯方块**: http://localhost:5173/tetris

如果这些页面能正常打开，说明集成完全成功，只是导航栏显示问题。

## 🔍 验证配置已加载

打开浏览器开发者工具（F12），切换到 Console，输入：

```javascript
document.querySelector('meta[name="config-version"]')?.content
```

如果返回 `"20260409-1752"`，说明最新配置已加载。

## ⚠️ 如果以上方法都无效

可能是 VitePress 的开发服务器有问题，尝试完全重启：

```bash
cd /root/.openclaw/workspace/blog

# 1. 停止所有相关进程
pkill -f vitepress

# 2. 清除所有缓存
rm -rf docs/.vitepress/cache
rm -rf node_modules/.vite
rm -rf dist

# 3. 重新启动
npm run dev
```

然后在浏览器中按 `Ctrl+Shift+R` 强制刷新。
