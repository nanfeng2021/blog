## 🚨 摔倒检测系统

**基于 3D 点云技术的实时监控系统**

### ⚠️ 重要提示
由于 Streamlit 框架的技术限制，本系统**无法通过域名路径访问**，需要通过**独立端口**访问。

### 功能特点
- 🔍 **实时监控**：20 FPS 流畅的 3D 点云动画
- 🎯 **自动识别**：检测人体姿态异常变化
- 🚨 **分级预警**：疑似摔倒 → 确认摔倒
- 📊 **数据统计**：FPS 趋势、处理耗时、报警记录

### 技术栈
- **前端框架**: Streamlit
- **3D 可视化**: Plotly + Open3D
- **数据处理**: NumPy, SciPy

### ✅ 访问方式

**方式 1（推荐）：直接通过 IP 和端口**
```
http://43.134.43.119:8501/
```

**方式 2：通过域名和端口**
```
http://ainanfeng.cn:8501/
```

> **为什么不能通过 `https://ainanfeng.cn/fall-detection/` 访问？**  
> Streamlit 使用相对路径加载资源（如 `./static/js/...`），当通过子路径访问时，浏览器会尝试从 `/fall-detection/static/...` 加载资源，但这些资源实际由 Streamlit 服务器内部管理，导致 404 错误。这是 Streamlit 框架的技术限制。

### 使用场景
- 养老院、医院病房监控
- 独居老人安全监护
- 工业场所人员安全监测
- 智能家居安防系统

[**立即体验 →**](http://43.134.43.119:8501/)

## 🏗️ 系统架构

```
┌─────────────────────────────────────┐
│         Nginx (Port 80)             │
│    统一入口 + 反向代理 + HTTPS      │
└──────────────┬──────────────────────┘
               │
    ┌──────────┼──────────┬──────────┬────────────┐
    │          │          │          │            │
    ▼          ▼          ▼          ▼            ▼
┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌──────────┐
│ 博客   │ │ 摔倒   │ │  RAG   │ │ 情感   │ │   AI     │
│ :8080  │ │ 检测   │ │ 知识库 │ │ 分析   │ │ 俄罗斯   │
│        │ │ :8501  │ │ :7860  │ │ :8001  │ │ 方块     │
│        │ │        │ │        │ │        │ │ :5000    │
└────────┘ └────────┘ └────────┘ └────────┘ └──────────┘
```

所有服务通过 Docker Compose 编排，Nginx 提供统一的访问入口和负载均衡。

## 🚀 快速启动

### 一键启动所有服务

```bash
cd /root/.openclaw/workspace/blog

# 使用 Docker Compose 启动
docker compose --profile with-rag --profile with-fall-detection up -d

# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f fall-detection
```

### 单独启动某个服务

```bash
# 只启动摔倒检测
docker compose --profile with-fall-detection up -d fall-detection

# 只启动 RAG 知识库
docker compose --profile with-rag up -d rag
```
